Логика ASK

1. Предложение

Система формирует pending-действие и сохраняет ask-state.

2. Подтверждение

Пользователь подтверждает или отменяет действие через текущий confirm flow.

3. Исполнение

После подтверждения действие должно идти через общий Core / Validation / Execution контур.
