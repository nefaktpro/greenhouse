# GREENHOUSE v15 — SYSTEM CONTEXT

## 1. Общая информация
- Проект: умная теплица на веранде.
- Архитектура: датчики → Smart Life → Home Assistant → Python → Telegram.
- Основная культура: клубника.
- Цель системы: мониторинг, интерпретация состояния, безопасное управление и дальнейшая автоматизация.

## 2. Конструкция
- Стеллаж двухъярусный.
- Два основных уровня: ВЕРХ и НИЗ.
- Среда неоднородная: разные микроклиматы по высоте, у окна и вдали от окна.
- Важны не только цифры, но и физический контекст зоны.

## 3. Верхний ярус
- Основная зона выращивания клубники.
- Освещение более мощное: верхний свет 300 Вт.
- Два ключевых горшка:
  - горшок у окна
  - дальний горшок
- Средняя влажность верхнего яруса считается по основным датчикам этих двух горшков.
- Для микроклимата верхнего яруса главный датчик — датчик у листьев.

## 4. Нижний ярус
- Нижняя зона стеллажа, тоже используется как зона выращивания клубники и тестовых кустов.
- Освещение менее интенсивное: 2×14 Вт.
- Три ключевых горшка:
  - серый
  - большой белый
  - круглый у окна
- Средняя влажность нижнего яруса считается по основным датчикам этих трёх горшков.
- Для микроклимата нижнего яруса главный датчик — датчик у листьев.

## 5. Роль окна
- Горшки у окна находятся в отдельном локальном контексте.
- У окна возможны:
  - холод
  - локальное пересыхание
  - перепады температуры
  - искажение “средней картины”
- Выводы по зоне у окна нужно делать осторожно.

## 6. Типы датчиков и их смысл
### Основные датчики
- Влажность грунта глубоко в горшке.
- Это главный ориентир для решений по поливу.

### Дополнительные датчики
- Поверхностная влажность.
- Температура поверхности.
- Воздух у корней.
- Они помогают понимать картину, но не заменяют основной датчик грунта.

### Климатические датчики
- Датчики у листьев.
- Общие датчики воздуха по ярусу.
- Датчики качества воздуха.

## 7. Полив
- Полив разделён на две независимые магистрали:
  - верх
  - низ
- У каждой магистрали:
  - насос
  - основной клапан
  - линия с удобрениями
  - защита от сухого хода
- Решение о поливе принимается по средним значениям основных датчиков влажности группы.
- Жёсткое ограничение: не более 60 секунд полива в день на ярус.
- Повторный полив — не раньше чем через 24 часа, если не будет отдельной ручной команды или пересмотра логики.

## 8. Удобрения
- Полив с удобрениями идёт по отдельной линии.
- Перед таким поливом должен работать насос перемешивания удобрений.
- ИИ должен знать, что это не обычный полив, а отдельный сценарий.

## 9. Увлажнение воздуха
- Увлажнитель работает через отдельную розетку питания и отдельный switch самого устройства.
- Ориентир по влажности:
  - целевой диапазон у листьев: 60–70%
- Главные датчики для решения:
  - верх: датчик у листьев
  - низ: датчик у листьев
- Увлажнитель нельзя держать включённым слишком долго подряд.

## 10. Вентиляция
- Есть отдельные вентиляторы для верхнего и нижнего ярусов.
- Они нужны для:
  - разгона воздуха
  - борьбы с застоем
  - снижения риска грибка
  - выравнивания температуры и влажности
- Важны при высокой влажности у листьев и при перегреве.

## 11. Свет
- Верхний свет — основной, мощный.
- Нижний свет — слабее, но тоже важен.
- Свет работает по расписанию.
- При авариях и пропаже питания свет отключается как не критичная нагрузка.

## 12. Штора
- Штора отделяет стеллаж от окна.
- Днём может открываться для естественного света и тепла.
- Ночью может закрываться для защиты от холода и возврата света обратно на растения.
- Для интерпретации климата у окна положение шторы важно.

## 13. Камеры
- Есть общий обзор и отдельные камеры по зонам.
- Камеры используются:
  - по запросу
  - для визуального контроля
  - для проверки выполнения автоматизаций
- Камеры не являются основным сенсором состояния, но важны для подтверждения реального эффекта.

## 14. Актуальность данных
- Актуальность данных — критическая часть смысла.
- Если данные устарели, их надёжность ниже.
- Число и свежесть данных должны анализироваться вместе.
- Старые данные нельзя трактовать как полностью надёжные.

## 15. Электропитание
- Есть основной щиток и резервный выключатель.
- Есть датчик наличия питания.
- При пропаже электричества нужно отключать всё лишнее и оставлять только критичные части системы.

## 16. Безопасность
### Пожар
- Пожарный контур — приоритет №1.
- При срабатывании пожарного датчика:
  1. сначала отключается щиток веранды
  2. затем резервный выключатель
- Все остальные сценарии в этот момент вторичны.

### Протечка
- При протечке нужно отключать весь полив и насосы.
- Возврат к работе только после ручной проверки.

### Пропажа электричества
- При пропаже сети отключаются энергоёмкие не критичные устройства.
- Система должна максимально долго сохранить связь, датчики и критические функции.

## 17. Главные датчики для интерпретации
- Верх, влажность у листьев: №65
- Верх, температура у листьев: №66
- Низ, влажность у листьев: №62
- Низ, температура у листьев: №63
- Основные датчики влажности грунта:
  - верх: №47, №56
  - низ: №32, №35, №41

## 18. Главные принципы интерпретации
- Основной датчик грунта важнее поверхностного.
- Датчики у листьев важнее общих датчиков воздуха.
- Воздух у корней не равен влажности грунта.
- Поверхность может быть сухой при влажной глубине.
- Зона у окна — отдельный локальный контекст.
- Верх и низ — разные микроклиматы.
- Нельзя делать сильный вывод по одному вторичному датчику без проверки контекста.
